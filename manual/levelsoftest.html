Levels of Testing

  1. Unit Testing
  2. Integration Testing
  3. System Testing
  4. User Acceptance Testing

1. Unit Testing

  1. A unit is a single component or module of a software
  2. Unit Testing conducts a single program or single module
  3. Unit Testing is Whitebox Testing technique
  4. Unit Testing conducted by the Developers
  5. Unit Testing Techniques
    • Basis Path Testing
    • Control Structure Testing
      • Conditional Coverage
      • Loops Coverage
    • Mutation Testing

2. Integration Testing

  1. Integration Testing is performed 2 or more modules
  2. Integration Testing is focuses on checking data communication between multiple modules.
  3. Integration Testing is White Box Testing technique.

Types of Integration Testing

  1. Incremental Integration Testing
  2. Non-Incremental Integration Testing

1. Incremental Integration Testing

  • Incrementally adding the modules and Testing the data flow between the modules
  • Incremental Integration Testing has three approaches
    1. Top-down Incremental Integration Testing
    2. Bottom-up Incremental Integration Testing
    3. Hybrid/Sandwich Apprach

1.1. Top-Down Incremental Integration Testing

Incrementally adding the modules and testing the dataflow between the modules. And ensure the module added is the child of previous module.

1.2. Bottom-up Incremental Integration Testing

Incrementally adding the modules and testing the dataflow between the modules. And ensure the module added is the parent of the previous module.

1.3. Hybrid/Sandwich Approach

Combination of Top-Down & Bottom-Up approach is called Sandwich/Hybrid Approach.

2. Non - Incremental Integration Testing/ Big Bang Testing

Adding all modules in a single shot and test data flow between modules.

Drawbacks
  • We might miss data flow between some of the modules.
  • If you find any defect, we can’t understand the root cause of defect.

3. System Testing

  1. Testing over all functionality of the application with respective client requirements.
  2. It is a black box testing technique.
  3. This testing is conducted by testing team.
  4. After completion of component and integration level testing's we start System testing.
  5. Before conducting system testing, we should know the customer requirements.
System Testing focuses on below aspects
  • User Interface Testing(GUI)
  • Usability Testing
  • Functional Testing
  • Non-Functinal Testing

4. User Acceptance Testing(UAT)

After completion of System Testing UAT team conducts acceptance testing in two levels

  • Alpha Testing : Alpha occurs after all features are developed, have been tested through the Development/ QA organizations, and have been signed off by the Product Owner.
  • Beta Testing : Beta Testing is type of user acceptance testing where the product team gives a nearly finished product to a group of target users to evaluate product performance in the real world.
