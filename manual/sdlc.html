SDLC

  1. SDLC stands for Software Development Life Cycle
  2. Software Development Life Cycle is a process used by Software Industry to design, develop and Test software's

  1. Requirement Analysis
  2. Feasibility Study
  3. Design
  4. Development/Coding
  5. Testing
  6. Deployment
  7. Maintainance

1.Requirement Analysis

  1. We have to collect and understand the requirement of the customer
  2. Normally BA's, Project Managers and Product owner are involved in this phase
  3. They will talk to the customer, get the requirements, and prepare a certain number of documents(BRS/SRS).
  4. This stage gives a clearer picture of the scope of the entire project and forecasts issues.
  5. This helps companies to finalize the necessary timeline to finish the work of that system.

Businsess Analyst

  1. BRS/BRD/CRD (Business Requirement Specification/Business Requirement Document/Customer Requirement Document
  2. Contains Functional Requirements in Non Technical Language

Sr. Team (System Analyst)

  1. SRS (System Requement Specification)
  2. Contains Functional Requirements in Technical Language.

2. Feasibility Study

  1. Product Manager
  2. BA (Business Analyst)
  3. Archtect
  4. Finance Team
  5. HR Team

Project Manager

Project Plan - Contains scope of the project, Budget, Resources, Team Schedule etc.

3. Design

  1. This helps to define the overall system architecture.
  2. In this phase, both system and software design documents are prepared as per the requirement specification document.
  3. This design phase serves as input for the next phase of the model. It contains documents like DDS (Design Document Specification) OR TDD (Technical Design Document)
  4. This prepares Architect/UI/UX Designer

Design Documents
1. HLD (High Level Document)

Describes flow between modules

  1. Brief description and name of each module.
  2. An outline of the functionality of every module.
  3. Integration between modules.
  4. Database table are identified along with their key elements.
  5. Complete architecture diagrams along with technical details.

2. LLD (Low Level Document)

Describes flow within the modules

  1. Actual Functional logic of the modules
  2. Complete input and outputs for every module.
  3. Complete details of the Integrations/interfaces.
  4. Addresses all types of dependency issues.
  5. List of error messages
  6. Database tables, which include type and size.

4. Development/Coding

  1. Once the system design phase is over, the next phase is coding.
  2. In this phase, developers start to build the entire system by writing code using the chosen programming language.
  3. In the coding phase, tasks ar divided into units or modules and assigned to the various developers.
  4. It is the longest phase of the Software Development Life Cycle process.
  5. In this phase, the developer needs to follow certain predefiened coding guidelines.

Developers

Build - Code is ready for Testing.

5. Testing

  1. Once the software is complete, It is deployed in the testing environment. The testing team starts testing the functinality of the entire system. This is done to verify that the entire application works according to the custormer's requirements.
  2. During this phase, QA and testing team may find some bugs/defects which they communicate to developers.
  3. Then development team fixes the bug and sends it back to QA for a retest.
  4. This process continues until the software is bug-free, stable and working according to the business needs of that system.

QA (Quality Analyst)

  1. Test Document - It is done by Tester STLC starts here
  2. STLC stands Software Testing Life Cycle.

6. Deployment

Field Engineer, Site Engineer

User Manual - Tested Software is deployed at client side.

7. Maintainance

Once the system is deployed, and customers start using the developed system, the following three activies may occur:

  1. Bug fixing : Bugs are reported because of some scenarios which are not tested at all.
  2. Upgrade: Upgrading the application to the newer versions of the Software.
  3. Enhancement: Adding some new features to the existing software.

If any defect is found by the end user or customer while using the product in live environment it will be handled by the maintainance team.

Types of SDLC

There are 2 types

  1. Incremental
  2. Iterative
